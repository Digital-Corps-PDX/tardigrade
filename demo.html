Introduction

Tardigrade provides standardized UI patterns for disclosing AI use in government services. Built on USWDS principles, these patterns help agencies meet transparency requirements while maintaining consistent, accessible interfaces.

These patterns complement AI Statements by providing the visual interface layer for AI transparency.

    Accessibility

    All patterns are WCAG 2.1 AA compliant and tested with screen readers. Key accessibility features include:

    • Proper ARIA labels for all interactive elements
    • Keyboard navigation support
    • Screen reader announcements for dynamic content
    • High contrast mode compatibility
    • Focus indicators that meet WCAG standards

    Testing Tip

    Test all AI disclosure patterns with screen readers to ensure users understand when AI is involved in decisions affecting them.

    Usage Guidelines

    When to Disclose AI Use

    • Always disclose when AI influences decisions about benefits, services, or rights
    • Always provide a path to human review for significant decisions
    • Always explain in plain language what the AI does and doesn't do
    • Never hide AI involvement behind technical jargon

    Contestability Requirements

    Every AI-influenced decision must include:

    1. Clear indication that AI was involved
    2. Option to request human review
    3. Expected timeline for review
    4. Contact method for questions
